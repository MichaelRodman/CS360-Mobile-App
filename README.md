# CS360-Mobile-App

## CS 360 Portfolio Artifact

This repository contains my completed Project Three artifact for CS 360. The artifact is a weight-tracking mobile application developed in Android Studio using Java. It includes the completed app code/design ZIP file, APK, and launch plan documents.

## Artifact Summary

The goal of this app was to create a simple and functional weight-tracking application for Android users. The app was designed to help users record their daily weight, set a goal weight, and monitor their progress over time. It also included an optional SMS notification feature that could alert the user when their goal weight was reached. The main user need this app addressed was having a quick, easy-to-use tool for tracking personal weight data in one place while keeping the process simple and private.

## Screens, Features, and User-Centered UI

To support user needs, the app included several main screens and features. These included a login screen, account creation, a dashboard screen, weight entry, editing and deleting weight entries, goal weight management, and optional SMS notifications. These features were necessary because users needed a way to securely access their data, enter information easily, review progress, and manage personal goals.

My UI designs kept users in mind by focusing on simplicity and clear navigation. Since weight tracking is something users may do daily, I wanted the app to be straightforward and not overcrowded. The layouts used clear labels, organized features logically, and kept the most important actions easy to access. I believe the designs were successful because they supported the main purpose of the app without adding unnecessary complexity.

## Coding Approach

My approach to coding this app was to work step by step and focus on one feature at a time. Instead of trying to build everything at once, I built individual pieces such as login functionality, database support, weight entry, and SMS handling, then tested each part as I went. This made it easier to identify problems and fix them before moving on.

One important strategy I used was breaking the project into smaller, manageable tasks. Another was reusing the UI design as a guide during development so that the coding stayed aligned with the intended user experience. These strategies can be applied in future projects because they help keep development organized, reduce confusion, and make debugging easier.

## Testing and Functionality

To test the app, I used the Android Emulator and also checked how the app behaved when values changed or when permissions were allowed or denied. I tested account creation, login, weight entry, editing and deleting entries, goal updates, and SMS-related functionality. I also used breakpoints and debugging tools in Android Studio to see where and when values were updating.

This process was important because testing helps confirm that the app actually works as intended and not just in theory. It also helps reveal bugs, design issues, and areas where the user experience may be unclear. Testing showed me that even when the core code works, there can still be issues with workflow, permissions, or how clearly certain features are presented to the user.

## Innovation and Challenges

One area where I had to innovate was making the app functional while keeping it simple. Adding features like login, persistent data storage, and optional SMS notifications created challenges because each feature had to work together without making the app confusing. I had to think carefully about how to organize screens and how to let the SMS feature remain optional so the app would still work even if permission was denied. That required balancing functionality with usability.

## Strongest Component

The component I was most successful with was building a functional app that connected user login, database storage, goal tracking, and optional SMS notifications into one complete experience. I think this showed my knowledge and skills well because it demonstrated more than just basic layout design. It showed that I could build an app with multiple connected features while keeping the user’s needs in mind.

## Reflection

This project helped me better understand the full mobile app development process from planning and UI design to coding, testing, and launch preparation. One of the biggest things I learned is that successful app development is not just about making code run. It also involves understanding the user, organizing features clearly, testing carefully, and improving the app through problem-solving. This project gave me a stronger foundation in mobile app development and helped me see how I can apply these skills in future courses and professional work.
